# 📌 专利标记助手

一款为专利代理人 / 审查员打造的桌面端 `.docx` 文档处理工具。围绕「撰写 → 自查 → 标注 → 生成」这条全链路，
把 **附图标记自动标注、文本清洗、错别字 / 重复字词检查、权利要求书引用审查** 集成到一个 PyQt6 界面里。

底层通过直接操作 `<w:t>` XML 节点实现**格式安全**：MathType / AxMath 公式、形状、图片、样式等对象
在所有自动化操作后都完整保留，不会被破坏。

## 四大功能 Tab 详解

### Tab 1：标记与标注

**做什么？** 把「齿圈」在正文里变成「齿圈（1）」或「齿圈1」，同时保留所有格式。

**界面要素**

- **附图标记编辑框** — 打开文档后自动用 `extract_marks_from_paragraphs` 扫描「附图说明」，
  支持跨多段落（含回车换行）拼接，识别 `1-齿圈，2-夹指，3-转盘` 这类常见格式并填入。
  识别不准时可直接手改再点「🔄 重新确认标记」。
- **可编辑文档预览** — 右侧预览区按段落逐行载入，支持就地编辑；改完点下方「✔ 确认修改」
  同步回 `doc_data['paragraphs']`，否则不影响原文。
- **⚡ 一键标注** — 权利要求书用带括号编号 `齿圈（1）`，具体实施方式用无括号编号 `齿圈1`，一次搞定。
- **仅标注权利要求书 / 仅标注具体实施方式** — 只处理单个章节。
- **🧹 一键清洗** — 移除全部附图编号，把文字还原。适合反复迭代修订时把之前加的编号清掉。

**操作步骤**

1. 拖入或选择 `.docx`
2. 看一眼「附图标记」里的字典是否全对；不对就改，改完点「重新确认标记」
3. 点「⚡ 一键标注」
4. 完成后切到任意其他 Tab 做下一步，或直接点右上角「💾 文件生成」

**为什么这么做不坏公式？**

`annotator.annotate_paragraph_safe()` 只修改 `<w:t>` 文本节点，不触碰 `<w:drawing>`、
`<m:oMath>` 这些对象节点。同一段落里夹着公式的文字替换也能正确对齐偏移。

---

### Tab 2：清洗与检测

**做什么？** 文档级的自动化清洗和非破坏性检测。

#### 🔹 删除「所述」
按章节勾选 `权利要求书 / 具体实施方式 / ...`，点按钮后把勾选章节里的所有「所述」一次性清空。

> 典型场景：权项一稿写满「所述」，改终稿前用这个功能批量清理。

#### 🔹 标点检查
- **半角→全角** — 把正文的 `,`、`.`、`;`、`?` 等替换为全角。
- **全角→半角（可选）** — 仅适用于特殊场景，默认关闭。
- **修正连续重复标点** — `，，` → `，`、`。。。` → `。`。

勾选项持久化，下次打开自动恢复上次选择。

#### 🔹 孤立附图标记检测（已增强）
这一功能会给出 **两类** 检测结果：

1. **孤立标记名** — 附图说明里写了 `齿圈`，但具体实施方式全篇都没提到 `齿圈`
2. **孤立图编号（新）** — 附图说明里写了 `图 5 为…`，但具体实施方式里没有出现 `图5` 字样

两类分别显示 ⚠️ 或 ✅ 状态。**该功能只检测、不修改文档**。

> 例：`附图说明` 提及 `图1~图5`，`具体实施方式` 只出现 `图3~图5` →
> 输出 `⚠️ 未引用图编号：图1、图2`

---

### Tab 3：错别字 / 重复字词检查

**做什么？** 词库 + NLP 双引擎找出文字错误，再交互式地逐条修正。

#### 🔹 引擎状态条
- **内置词库引擎** — 显示当前加载了多少条规则，点击直达词库编辑器。
- **离线 NLP 引擎（pycorrector）** — 入口已移至词库编辑器的三级菜单「专业用户」，避免给新用户造成困扰。
  软件内有零基础的「一键安装 → 模型下载 → 验证」教程卡片。

#### 🔹 错别字检查
1. 点「🔎 开始错别字检查」
2. 结果表格列：`位置 | 原文片段 | 建议修改 | 操作`
3. 「建议修改」列可直接编辑
4. 不想采用的条目点行内「忽略」（无底色无边框，仅文字，与表格同色系）
5. 点「💾 应用修改」把所有建议批量写回文档；写回完成后表格自动清空，
   作为「本轮所有可修正条目均已应用」的视觉确认

词库可在「词库编辑器」里增删任意条目（含内置），修改立即生效。

#### 🔹 重复字词检查
同一个结果表格，共用同一套工作流。典型会被查出的：`所述所述`、`发明发明`、`阶段性阶段性` 等。

「忽略词库」按钮打开 **4 列网格编辑器**：
- 搜索 / 快速添加 / 删除
- 导入 / 导出 JSON
- 加入后再次「开始检查」时自动跳过

### Tab 4：权利要求书检查

**这是最大的新功能**，针对权利要求书的六项专业审查，配可编辑预览 + 联动高亮。

#### 🔹 顶部工具栏

```
[检查字数]  [2] [3] [4] [5] [6] [7]  |  自定义 [ 8 ]  [🔎 开始检查]  [📝 不确定用语词库]
```

- **检查字数** — 对「引用基础」和「术语一致性」两项按 N 字滑窗扫描。
  2~7 点选即用；需要更大的 N 时用右侧「自定义」输入框（2~30，输入框宽度随数字位数自适应，
  不带上下箭头按钮）。
  其余 4 项检查与 N 无关，点「开始检查」时一并跑。
- **🔎 开始检查** — 从当前预览框内容（!）切段，调 `claim_check.run_all_checks()`。
- **📝 不确定用语词库** — 30 条内置 + 4 列网格编辑器。

#### 🔹 左右可拖分割面板

**左侧：可编辑预览** — `QPlainTextEdit`，初始化时把权利要求书章节每段一行载入。
可以直接在里面改错别字 / 换术语；改完点下方 **「✔ 确认修改」** 才会同步回 `doc_data['paragraphs']`。
未确认的修改会在状态栏提示「⚠️ 有未确认修改」并把按钮高亮。

**生成文件时的保护**：若存在未确认修改，点「💾 文件生成」会弹对话框问「是否立即确认并生成」，
确认后才继续走 annotator 流水线。

**右侧：检查结果表** — 5 列：`类型 | 权项 | 上下文 | 说明 | 操作`
- **双击「上下文」** — 左侧预览框自动跳到对应段落，并用柔和黄背景 + 深蓝文字高亮关键片段
- **双击「说明」**（新）— 弹出只读详情对话框，完整显示 `类型 / 位置 / 上下文 / 说明 / 建议`，不再被截断
- **「忽略」按钮** — 本次会话忽略（不写入任何持久词库），同类型同术语的行会一并移除

#### 🔹 六项检查内容

| 编号 | 类型 (kind) | 检查目标 | 触发 |
|---|---|---|---|
| ① | `antecedent` | **引用基础** — `所述 X` 首次出现前权项是否已定义过 `X` | N 字滑窗 |
| ② | `dependency` | **引用关系** — `根据权利要求 X 所述` 的 X 是否存在 / 是否前引 / 是否自引 / 是否循环 | 每次必跑 |
| ③ | `term` | **术语一致性** — 同一权利要求书内 `所述齿圈` vs `所述齿环` 等差异 | N 字滑窗，仅扫 `所述` 后面的术语 |
| ④ | `vague` | **不确定用语** — `约`、`大概`、`可能`、`优选`、`左右`… | 每次必跑，对照「不确定用语词库」 |
| ⑤ | `numbering` | **独立权项序号** — 独立权项的编号是否从 1 开始且连续 | 每次必跑 |
| ⑥ | `multi_dep` | **多项引用合法性** — `权利要求 1、2 或 3` 这种「或」引用的位置与格式 | 每次必跑 |

每条结果返回字段：`{kind, claim_no, para_idx, context, message, suggestion}`。

#### 🔹 不确定用语词库（📝 按钮）

- 4 列网格显示 30 条内置默认词（约 / 大概 / 大致 / 优选 / 左右 / 大体 / 基本 / 大约 / 略 / 近似 / …）
- 支持搜索、添加、删除、导入、导出（JSON / TXT）
- 「🔄 恢复内置」按钮把缺失的内置词合并回当前列表（不会删除用户已有条目）
- 保存后下次「开始检查」立即生效

#### 🔹 操作步骤

```
1. 打开 docx（左侧预览自动载入权利要求书）
2. 选择「检查字数」— 默认 2
3. 点「🔎 开始检查」
4. 右侧结果表按类型排序出现
5. 双击「上下文」定位 → 在左侧直接改
6. 改完点「✔ 确认修改」
7. 再点「🔎 开始检查」重扫，直到清零
8. 右上「💾 文件生成」输出最终 docx
```

## 典型工作流

**场景一：刚从代理人拿到初稿，要做一轮全面自查**

1. 拖入 docx
2. Tab 3「错别字检查」→ 应用修改
3. Tab 3「重复字词检查」→ 应用修改
4. Tab 2「孤立标记检测」→ 看看有没有图没引用
5. Tab 4「权利要求书检查」→ N=2 点开始 → 逐条处理
6. Tab 1「⚡ 一键标注」
7. 「💾 文件生成」→ 输出 `xxx_已标注.docx`

**场景二：只想快速给具体实施方式加标注**

1. 拖入 docx
2. 看一眼附图标记编辑框对不对，不对就改
3. 点「仅标注具体实施方式」
4. 「💾 文件生成」

**场景三：审查意见回函前的术语一致性复盘**

1. 拖入最新稿
2. Tab 4 → N 调到 3 或 4 → 点开始检查
3. 看 `term` 类结果，双击上下文定位
4. 左侧改名统一 → 确认修改 → 重扫
5. 生成文件

## 用户数据位置

所有用户级配置和自定义词库通过 `QSettings` + JSON 持久化，存放在：

```
C:\Users\<用户名>\AppData\Local\PatentMarker\MarkAssistant\
├── user_wordbank.json              # 自定义错别字词库
├── disabled_builtin_wordbank.json  # 已禁用的内置词条
├── dup_ignore_wordbank.json        # 重复字词忽略词库
└── vague_wordbank.json             # 不确定用语词库
```

界面设置（主题、勾选状态、窗口几何、最近目录）存放在 Windows 注册表的
`HKCU\Software\PatentMarker\MarkAssistant` 下，由 `QSettings` 自动管理。

换电脑时把这两处一起迁移即可带走所有自定义。

## 常见问题

**Q：标注后公式 / 图形消失了怎么办？**
A：不会消失。`annotate_paragraph_safe` 只动 `<w:t>` 节点，不改 `<w:drawing>` / `<m:oMath>`。如果你真的遇到这种情况，请附最小复现文件反馈。

**Q：权利要求书检查结果里「术语一致性」误报怎么处理？**
A：术语检测只扫描 `所述` 后面的术语串（降噪策略）。仍有误报时：
1. 点行内「忽略」— 本次会话生效，不入词库
2. 或把 N 调大（如 3 / 4）获得更长的术语锚点

**Q：预览框改了行数会怎样？**
A：v1 严格要求行数等于原段落数。行数不一致点「确认修改」时会弹警告并拒绝同步，建议改用 Tab 2 的「删除所述」等批量功能，或调回原始行数。

**Q：不确定用语词库被我删空了，怎么恢复？**
A：点编辑器里的「🔄 恢复内置」按钮，把内置 30 条默认词合并回当前列表。

**Q：生成文件时提示「权利要求书有未确认修改」？**
A：切回 Tab 4，点「✔ 确认修改」或直接在弹窗里选「立即确认并生成」。这是防止预览框里手改的内容因忘记确认而遗失。

**Q：怎么禁用某条内置错别字规则？**
A：打开错别字词库编辑器，选中要禁用的条目点「🚫 禁用」。禁用状态持久化保存，随时可重新启用。
