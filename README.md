# 📌 专利标记助手 V4.1.1

一款为专利代理人 / 审查员打造的桌面端 `.docx` 处理工具。围绕「撰写 → 自查 → 标注 → 生成」全链路，
把 **附图标记自动标注、文本清洗、错别字 / 重复字词检查、权利要求书引用审查、说明书检查** 集成到一个 PyQt6 界面里。

**V4.0.0 整体重构为三区布局**：专利正文成为常驻**主舞台**并支持**结构化实时编辑**；图片与公式**内联富显示**，
检查问题在原文上**内联高亮**，结果与正文一屏对照。底层仍通过直接操作 `<w:t>` XML 节点实现**格式安全**：
MathType / AxMath 公式、形状、图片、样式等对象在所有自动化操作后都完整保留，不会被破坏。

## 下载与安装

发行包位于 [Releases](https://github.com/vvangpc/mark123/releases) 页面，两种形式任选：

### 🅰️ 安装版（推荐）`mark123-Setup-vX.Y.Z.exe` · 约 32 MB

- 双击运行 → 向导中**可自选安装目录** → 自动创建开始菜单 / 桌面快捷方式 → 控制面板可一键卸载
- 勾选「文件关联」后，**右键任意 `.docx` 文件即可看到「用专利标记助手打开」**，点击后软件启动并自动导入该文档，可直接一键检查

> 右键菜单走 `SystemFileAssociations\.docx\shell` 注册，不会抢占 Word 的默认打开行为，仅追加一个菜单项；卸载时自动清理注册表。

### 🅱️ 单文件版 `专利标记助手-单文件版.exe` · 约 44 MB

- 免安装，复制到任意位置双击即可运行
- 启动时需解压到 `%TEMP%`，首次启动略慢（5–15 秒，带启动图遮挡）

两个版本功能完全一致，选一个即可。

> 🔄 **内置在线更新**：每次启动会自动从 GitHub Releases 检查新版本（HTTPS），
> 发现新版后弹窗提示；点「立即更新」即自动下载安装包、校验 SHA256 后拉起安装程序。
> 也可在右下角「⚙ 设置 → 🔄 检查更新」手动触发。更新源全程走 GitHub，无需任何自建服务器。

## 主界面结构（三区布局）

无顶部标题栏，最大化内容与操作面积。左侧是「看与改」的主舞台，右侧是「选模块 → 点功能」的导航与文件操作。

```
┌──────────────────────────────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────┐  ┌──────┬─────────────┐  │
│  │ 1框 · 内容主舞台（常驻 · 可编辑）         │  │ 3列  │   4列        │  │
│  │ ┌─────────────────────────────────────┐ │  │ 模块 │ 子功能 / 操作 │  │
│  │ │权项│说明书│附图│摘要│日志│历史 ←标签│ │  │📌标记│  ……          │  │
│  │ ├─────────────────────────────────────┤ │  │🧹清洗│              │  │
│  │ │ 专利正文：内联图片 / 公式，          │ │  │📝错别字│            │  │
│  │ │ 错别字·重复字·权项 问题内联高亮       │ │  │🔁重复字│            │  │
│  │ │ （就地编辑，✓ 确认修改 回写内存）     │ │  │⚖️权项 │             │  │
│  │ └─────────────────────────────────────┘ │  └──────┴─────────────┘  │
│  ├─────────────────────────────────────────┤  ┌─────────────────────┐ │
│  │ 2框 · 当前子功能面板                      │  │ 📂 文件上传（跑马灯）│ │
│  │ （结果表 / 标记字典 / 清洗卡片 /          │  │ 💾 文件生成          │ │
│  │   全文替换输入框 —— 随模块切换）          │  │ ☐ 打开目录    ⚙ 设置 │ │
│  └─────────────────────────────────────────┘  └─────────────────────┘ │
├──────────────────────────────────────────────────────────────────────┤
│ 状态栏：当前状态消息                                      [进度条]      │
└──────────────────────────────────────────────────────────────────────┘
  ↕ 三处可拖拽分隔条：1框↔2框、左区↔右区、3列↔4列；2框 可向下拖动收起
```

四个区域有固定简称，便于定位：

- **1框** —— 左上**内容主舞台**（常驻）。六个标签页：`权利要求书 / 说明书 / 说明书附图 / 说明书摘要 / 操作日志 / 操作历史`。专利正文在此查看、就地编辑、看内联图片 / 公式与内联高亮；约占左侧 2/3 高度，**不可收起**。
- **2框** —— 左下**当前子功能面板**，随右侧选中的模块切换内容（结果表 / 附图标记字典 / 清洗卡片 / 全文替换输入框）。可向下拖动收起。
- **3列** —— 右侧**第一列**，固定六个模块：`标记 / 清洗 / 错别字 / 重复字 / 权项 / 说明书`。
- **4列** —— 右侧**第二列**，随 3列 选择动态变化：或列出该模块的子功能，或直接放该模块的操作按钮。

右侧列底部依次是：**文件上传按钮**（兼当前文件名提示，长名跑马灯滚动）、**💾 文件生成**、以及一行「☐ 打开目录 + ⚙ 设置」。
通过 Windows 右键菜单或拖入打开 docx 时自动载入；加载后六个模块同时就绪、互不影响，最后点「💾 文件生成」输出结果。

## 六大模块详解

> 交互通用规律：**3列 选模块 → 4列 点功能 → 2框 看结果 / 填参数 → 1框 看正文与高亮**。

### 📌 标记

把「齿圈」在正文里变成「齿圈（1）」或「齿圈1」，同时保留所有格式。

- **2框** —— 「附图标记字典」编辑框：打开文档后自动扫描「附图说明」填入；识别不准可直接手改。
- **4列** —— 操作按钮分三组：
  - *附图标记字典*：`✅ 重新确认标记`（把字典写回内存）/ `🔄 重新提取标记`（从原文重抽）
  - *批量标注*：`⚡ 一键标注`（权项 + 具体实施方式）/ `📋 仅标注权利要求书` / `📝 仅标注具体实施方式`
  - *清除标记*：`🧹 删除所有标记` / `🗑️ 清空标记`
- **只导入单独的说明书**（无权利要求书）时，用「仅标注具体实施方式」标注、「删除所有标记」清除即可。

### 🧹 清洗

- **4列** 列出四项功能，点击在 2框 展开对应卡片 / 输入框：
  - `🗑️ 删除「所述」` —— 按章节勾选后一次清空
  - `🔣 标点检查` —— 半角↔全角、修正 `，，` `。。。`（执行顺序：半角→全角 → 全角→半角 → 修正连续重复）
  - `🔍 孤立标记检测` —— 同时输出「孤立标记名」和「孤立图编号」两类，仅检测不修改
  - `🔁 全文替换`（V4 新）—— 在 2框 填「替换前 / 替换后」，点 4列「✅ 确认替换」，格式安全，保留公式 / 图片

### 📝 错别字 / 🔁 重复字

两个独立模块，**共用 2框 同一结果表**（标题随当前检查类型动态切换）。

- **4列** —— `🔍 检查`（主操作）/ `✅ 应用所有修改`（把「修改后」列批量写回）/ 词库按钮（错别字词库 · 重复字忽略词库，均可增删 / 导入 / 导出）。
- **2框 结果表** 五列：`章节 | 修改前 | 修改后 | 修改 | 忽略`——「修改后」可直接编辑；点「修改」应用单条、点「忽略」跳过本条。
- **1框 内联高亮**：检查后所有问题在正文**标黄**；单击「修改前」跳转该处并**标红**。错别字内置 165+ 条词库 + 用户自定义；重复字识别 `所述所述`、`AA`、`ABCABC` 等连续重复。

### ⚖️ 权项

六项专业审查，配 1框 内联高亮 + 单击定位。

| # | 类型 | 检查目标 |
|---|---|---|
| ① | `antecedent` | **引用基础** —— `所述 X` 首次出现前是否已定义 X（支持同权项内位置先后比对） |
| ② | `dependency` | **引用关系** —— `根据权利要求 X 所述` 的 X 是否存在 / 前引 / 自引 / 循环 |
| ③ | `vague` | **不确定用语** —— `约` / `大概` / `优选` / `左右` 等（对照词库，可关） |
| ④ | `numbering` | **独立权项序号** —— 编号是否从 1 开始且连续 |
| ⑤ | `multi_dep` | **多项引用合法性** —— `权利要求 1、2 或 3` 这种「或」引用 |
| ⑥ | `ending` | **句号结尾** —— 每条权利要求必须以「。」结尾 |

- **4列 参数**：`检查字数`（2~7 预设 + 自定义 2~30，仅影响 ①③）/ `动态截断[黑名单]` / `动态回退`（引用基础降噪）/ `不确定用语检查[词库]` / `▶ 开始检查`。
- **2框 结果表** 四列：`类型 | 权项 | 说明 | 操作`——可定位的问题已在 1框「权利要求书」标黄，单击「说明」跳到 1框 并把该条标红；「操作」可忽略本条。
- **前言保护**：权项首段 `一种…，其特征在于` 前的专利名称不参与标注与引用基础检测。
- 在 1框 改完正文（记得「✓ 确认修改」）后再次「开始检查」即重扫。

### 📑 说明书（V4.1 新）

针对说明书 / 具体实施方式的只读检查模块，**共用 2框 结果表**（标题随检查类型动态），可逐步接入更多检查。

- **4列** —— `📑 实施例编号`（主操作）/ `📊 摘要字数`；按文档章节存在性自动启用。
- **📑 实施例编号** —— 扫描「具体实施方式」，校验实施例标题应为 实施例一、实施例二、实施例三…**从「一」起、连续、不重复、不颠倒**；缺号 / 重号 / 顺序颠倒 / 起始非一都会报出。兼容 `实施例一` / `实施例1` / `第一实施例` / `实施方式一`（只看序号，不挑数字风格），并能区分标题与行内引用（「如实施例一所述」不计入）。
- **📊 摘要字数** —— 「说明书摘要」文字部分超过细则规定的 **300 字** 时提示。
- **2框 结果表** 四列：`类型 | 定位 | 说明 | 操作`——可定位问题已在 1框 标黄（实施例 → 说明书标签，摘要 → 说明书摘要标签），单击「说明」跳转标红，「操作」忽略本条。

## 内联富显示与格式安全

### 内联富显示（V4 新）

含附图 / 公式的标签页改用富文本构建：**按段内文档顺序**插入文本与对象——

- **图片**：附图 PNG / JPEG 直接解码显示。
- **公式**：MathType / AxMath 的 WMF / EMF 预览经 Windows GDI（`ui/render/formula.py`，纯 `ctypes` 零依赖）转成位图显示；OMML 公式以「【公式】」占位。
- 每段仍是**一个段落 block、内联对象是块内占位符（U+FFFC）不增行**，因此「行＝段」的编辑 / 回写 / 高亮模型完全不变。
- 窗口缩放时按 1框 宽度防抖重排，图片走缓存不重复解码。

> 含图片 / 公式的段落在 1框 内为**只读**，避免误改对象；纯文本段可自由编辑。

### 为什么不破坏格式

所有自动化（标注 / 清洗 / 替换 / 错字回写）都只修改 `<w:t>` 文本节点，
完整跳过 `<w:drawing>`、`<m:oMath>`、`<w:object>` 等对象节点。
核心约束：**结构化编辑禁止增删段落**（保证段落索引稳定），**含对象段只读、回写只认 `<w:t>`**。
因此公式、形状、图片、样式在标注前后逐字节保留。

**场景一：从代理人手里拿到初稿，做一轮全面自查**

1. 右键 docx →「用专利标记助手打开」（或拖入窗口）
2. **错别字** → 检查 → 看 1框 标黄逐条核对 → 应用所有修改
3. **重复字** → 检查 → 应用所有修改
4. **清洗 → 孤立标记检测** → 看有无图没引用
5. **权项** → 检查字数 N=2 →「开始检查」→ 单击「说明」跳 1框 逐条处理
6. **标记** →「⚡ 一键标注」
7. 「💾 文件生成」→ 输出 `xxx_已标注.docx`

**场景二：只想快速给具体实施方式加标注**

1. 拖入 docx
2. 在「标记」模块核对 2框 的附图标记字典
3. 4列 点「📝 仅标注具体实施方式」→「💾 文件生成」

## 从源码运行

需要 Python 3.10+：

```bash
pip install PyQt6 python-docx lxml
python main.py
```

或用 `uv`：

```bash
uv sync
uv run python main.py
```

命令行也支持直接传入文件路径：`python main.py path/to/doc.docx`（软件启动后自动载入）。

> 内联公式显示依赖 Windows GDI（`gdi32` / `user32`），仅在 Windows 下可用；其它平台公式回退为占位符，其余功能不受影响。

## 用户数据位置

所有用户级词库以 JSON 形式存放在：

```
C:\Users\<用户名>\AppData\Local\PatentMarker\MarkAssistant\
├── user_wordbank.json              # 自定义错别字词库
├── disabled_builtin_wordbank.json  # 已禁用的内置词条
├── dup_ignore_wordbank.json        # 重复字词忽略词库
└── vague_wordbank.json             # 不确定用语词库
```

界面设置（主题、勾选状态、窗口几何、最近目录）存放在注册表
`HKCU\Software\PatentMarker\MarkAssistant`，由 `QSettings` 自动管理。

换电脑时把这两处一起迁移即可带走所有自定义。

## 自行打包

### 第一步：PyInstaller 打包 exe

```bash
python build.py            # onedir 模式（启动 1–2 秒，推荐）
python build.py onefile    # onefile 单文件（启动 5–15 秒）
```

产物：

- **onedir**：`dist/专利标记助手/` 文件夹（含 `专利标记助手.exe` + 依赖）
- **onefile**：`dist/专利标记助手.exe` 单文件

### 第二步（可选）：Inno Setup 打包安装版

先确保已经跑过 `python build.py onedir` 生成 `dist/专利标记助手/`，然后：

1. 安装 [Inno Setup 6](https://jrsoftware.org/isdl.php) 或 `winget install JRSoftware.InnoSetup`
2. 编译脚本：

```bash
iscc installer.iss
# 或在 Inno Setup Compiler 里打开 installer.iss → Build
```

产物：`dist/mark123-Setup-vX.Y.Z.exe`（约 32 MB），具备：

- 向导式自选安装目录
- 开始菜单 / 桌面快捷方式
- 可选 `.docx` 右键菜单集成
- 控制面板一键卸载（含注册表清理）

> 📦 **正式发版**走 GitHub Actions：推送 `v4.0.0` 形式的 git tag 即触发 CI 自动改写 `version.py`、
> 跑 `build.py` + `iscc`，并把安装包与 `latest.json` 发布到 GitHub Releases（供在线更新校验）。

## 常见问题

**Q：标注后公式 / 图形消失了怎么办？**
A：不会消失。所有自动化只动 `<w:t>` 节点，不改 `<w:drawing>` / `<m:oMath>`。如真遇到，请附最小复现文件反馈。

**Q：1框 里某段改不动 / 是只读的？**
A：含图片或公式的段落为只读，避免误删对象。请在该段以外编辑；纯文字段落可自由修改，改完点标签栏右上角「✓ 确认修改」回写内存。

**Q：为什么不能在 1框 里回车新增 / 删除整段？**
A：结构化编辑以「行＝段」维持段落索引稳定，禁止增删段落（否则定位 / 高亮会全盘错位）。增删段落请回 Word 处理后重新导入。

**Q：深色主题下公式是白底小方块？**
A：公式预览由 WMF / EMF 渲染在白底上，深色主题下会有反差但内容清晰可读，属已知小瑕疵。

**Q：右键菜单「用专利标记助手打开」没有出现？**
A：安装时确认勾选了「文件关联」任务；或重新运行安装版选择「修复」；或在 `HKCU\Software\Classes\SystemFileAssociations\.docx\shell\` 下确认 `专利标记助手` 键存在。

**Q：生成文件时提示「有未确认修改」？**
A：回 1框 点「✓ 确认修改」，或在弹窗里选「立即确认并生成」。这是防止 1框 手改内容因忘记确认而遗失。

**Q：不确定用语词库被删空了，怎么恢复？**
A：点编辑器里的「🔄 恢复内置」按钮，把内置词合并回当前列表。

**Q：单文件版首次启动特别慢？**
A：onefile 模式需要把内嵌文件解压到 `%TEMP%\_MEIxxx`，首启需 5–15 秒；之后再启动稍快。追求最快启动请用**安装版**（onedir 模式，1–2 秒即启）。
