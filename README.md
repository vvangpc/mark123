# 📌 专利标记助手 V3.5

一款为专利代理人 / 审查员打造的桌面端 `.docx` 处理工具。围绕「撰写 → 自查 → 标注 → 生成」全链路，
把 **附图标记自动标注、文本清洗、错别字 / 重复字词检查、权利要求书引用审查** 集成到一个 PyQt6 界面里。

底层通过直接操作 `<w:t>` XML 节点实现**格式安全**：MathType / AxMath 公式、形状、图片、样式等对象
在所有自动化操作后都完整保留，不会被破坏。

## 下载与安装

发行包位于 [Releases](https://github.com/vvangpc/mark123/releases) 页面，两种形式任选：

### 🅰️ 安装版（推荐）`专利标记助手V3.5-安装版.exe` · 约 32 MB

- 双击运行 → 向导中**可自选安装目录** → 自动创建开始菜单 / 桌面快捷方式 → 控制面板可一键卸载
- 勾选「文件关联」后，**右键任意 `.docx` 文件即可看到「用专利标记助手打开」**，点击后软件启动并自动导入该文档，可直接一键检查

> 右键菜单走 `SystemFileAssociations\.docx\shell` 注册，不会抢占 Word 的默认打开行为，仅追加一个菜单项；卸载时自动清理注册表。

### 🅱️ 单文件版 `专利标记助手V3.5-单文件版.exe` · 约 44 MB

- 免安装，复制到任意位置双击即可运行
- 启动时需解压到 `%TEMP%`，首次启动略慢（5–15 秒，带启动图遮挡）

两个版本功能完全一致，选一个即可。

## 四大功能 Tab 详解

### Tab 1：标记与标注

把「齿圈」在正文里变成「齿圈（1）」或「齿圈1」，同时保留所有格式。

- **附图标记编辑框** — 打开文档后自动扫描「附图说明」段落并填入；识别不准可直接手改再点「🔄 重新确认标记」
- **可编辑文档预览** — 右侧按段落逐行载入，支持就地编辑；改完点「✔ 确认修改」同步回内存数据
- **⚡ 一键标注** — 权利要求书 `齿圈（1）` + 具体实施方式 `齿圈1`
- **仅标注权利要求书 / 仅标注具体实施方式** — 只处理单个章节
- **🧹 一键清洗** — 移除全部附图编号，恢复纯文字

> 为什么不坏公式？`annotator.annotate_paragraph_safe()` 只修改 `<w:t>` 文本节点，完整跳过 `<w:drawing>`、`<m:oMath>` 等对象节点。

### Tab 2：清洗与检测

- **删除「所述」** — 按章节勾选后一次清空
- **标点检查** — 半角↔全角、修正 `，，` `。。。`
- **孤立附图标记检测** — 同时输出「孤立标记名」和「孤立图编号」两类，仅检测不修改

### Tab 3：错别字 / 重复字词检查

- 词库引擎 + 可选离线 NLP（`pycorrector`，见软件内教程）
- 结果表 4 列：`位置 | 原文 | 建议修改 | 操作`；建议可直接编辑
- 「忽略」列点击即跳过本条
- 「💾 应用修改」批量写回；重复字词共用同一工作流

### Tab 4：权利要求书检查

六项专业审查，配可编辑预览 + 联动高亮。

| # | 类型 | 检查目标 | 触发 |
|---|---|---|---|
| ① | `antecedent` | **引用基础** — `所述 X` 首次出现前是否已定义 X（支持同权项内位置先后比对） | N 字滑窗 |
| ② | `dependency` | **引用关系** — `根据权利要求 X 所述` 的 X 是否存在 / 前引 / 自引 / 循环 | 每次必跑 |
| ③ | `term` | **术语一致性** — `所述齿圈` vs `所述齿环` 等差异 | N 字滑窗，仅扫 `所述` 后面 |
| ④ | `vague` | **不确定用语** — `约` / `大概` / `优选` / `左右` 等 | 每次必跑（对照词库） |
| ⑤ | `numbering` | **独立权项序号** — 编号是否从 1 开始且连续 | 每次必跑 |
| ⑥ | `multi_dep` | **多项引用合法性** — `权利要求 1、2 或 3` 这种「或」引用 | 每次必跑 |

- **左侧可编辑预览** — `QPlainTextEdit`，每段一行；改完点「✔ 确认修改」才同步回 `doc_data`
- **右侧结果表** — 双击「上下文」跳转高亮；双击「说明」弹完整详情；行内「忽略」本次会话生效
- **字数选择** — 2~7 快捷按钮 + 自定义 2~30；仅影响 ①③ 两项
- **不确定用语词库** — 30 条内置 + 4 列网格编辑，可增删 / 导入 / 导出 / 恢复内置
- **前言保护** — 权项首段 `一种…，其特征在于` 前的专利名称部分不参与标注与引用基础检测

## 典型工作流

**场景一：从代理人手里拿到初稿，做一轮全面自查**

1. 右键 docx →「用专利标记助手打开」
2. Tab 3「错别字检查」→ 应用修改
3. Tab 3「重复字词检查」→ 应用修改
4. Tab 2「孤立标记检测」→ 看有无图没引用
5. Tab 4「权利要求书检查」→ N=2 点开始 → 逐条处理
6. Tab 1「⚡ 一键标注」
7. 「💾 文件生成」→ 输出 `xxx_已标注.docx`

**场景二：只想快速给具体实施方式加标注**

1. 拖入 docx
2. 核对附图标记编辑框
3. 点「仅标注具体实施方式」→「💾 文件生成」

**场景三：审查意见回函前的术语一致性复盘**

1. 打开最新稿 → Tab 4 → N 调到 3 或 4 → 开始检查
2. 看 `term` 类结果，双击上下文定位
3. 左侧改名统一 → 确认修改 → 重扫 → 生成

## 从源码运行

需要 Python 3.10+：

```bash
pip install PyQt6 python-docx lxml
python main.py
```

或用 `uv`：

```bash
uv sync
uv run python main.py
```

命令行也支持直接传入文件路径：`python main.py path/to/doc.docx`（软件启动后自动载入）。

## 用户数据位置

所有用户级词库以 JSON 形式存放在：

```
C:\Users\<用户名>\AppData\Local\PatentMarker\MarkAssistant\
├── user_wordbank.json              # 自定义错别字词库
├── disabled_builtin_wordbank.json  # 已禁用的内置词条
├── dup_ignore_wordbank.json        # 重复字词忽略词库
└── vague_wordbank.json             # 不确定用语词库
```

界面设置（主题、勾选状态、窗口几何、最近目录）存放在注册表
`HKCU\Software\PatentMarker\MarkAssistant`，由 `QSettings` 自动管理。

换电脑时把这两处一起迁移即可带走所有自定义。

## 自行打包

### 第一步：PyInstaller 打包 exe

```bash
python build.py            # onedir 模式（启动 1–2 秒，推荐）
python build.py onefile    # onefile 单文件（启动 5–15 秒）
```

产物：

- **onedir**：`dist/专利标记助手/` 文件夹（含 `专利标记助手.exe` + 依赖）
- **onefile**：`dist/专利标记助手.exe` 单文件

### 第二步（可选）：Inno Setup 打包安装版

先确保已经跑过 `python build.py onedir` 生成 `dist/专利标记助手/`，然后：

1. 安装 [Inno Setup 6](https://jrsoftware.org/isdl.php) 或 `winget install JRSoftware.InnoSetup`
2. 编译脚本：

```bash
iscc installer.iss
# 或在 Inno Setup Compiler 里打开 installer.iss → Build
```

产物：`dist/专利标记助手V3.5-安装版.exe`（约 32 MB），具备：

- 向导式自选安装目录
- 开始菜单 / 桌面快捷方式
- 可选 `.docx` 右键菜单集成
- 控制面板一键卸载（含注册表清理）

## 常见问题

**Q：标注后公式 / 图形消失了怎么办？**
A：不会消失。`annotate_paragraph_safe` 只动 `<w:t>` 节点，不改 `<w:drawing>` / `<m:oMath>`。如真遇到，请附最小复现文件反馈。

**Q：权利要求书检查结果里「术语一致性」误报怎么处理？**
A：该检测只扫描 `所述` 后面的术语串。仍有误报时：① 行内「忽略」本次会话生效；② 把 N 调大（3 / 4）获得更长锚点。

**Q：右键菜单「用专利标记助手打开」没有出现？**
A：安装时确认勾选了「文件关联」任务；或重新运行安装版选择「修复」；或在 `HKCU\Software\Classes\SystemFileAssociations\.docx\shell\` 下确认 `专利标记助手` 键存在。

**Q：生成文件时提示「权利要求书有未确认修改」？**
A：切回 Tab 4 点「✔ 确认修改」，或在弹窗里选「立即确认并生成」。这是防止预览框手改内容因忘记确认而遗失。

**Q：不确定用语词库被删空了，怎么恢复？**
A：点编辑器里的「🔄 恢复内置」按钮，把 30 条内置词合并回当前列表。

**Q：单文件版首次启动特别慢？**
A：onefile 模式需要把内嵌文件解压到 `%TEMP%\_MEIxxx`，首启需 5–15 秒；之后再启动稍快。追求最快启动请用**安装版**（onedir 模式，1–2 秒即启）。
